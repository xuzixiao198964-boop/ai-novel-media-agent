## 1. 集成测试策略

### 1.1 测试层次

```
┌─────────────────────────────────────────────────┐
│               E2E 测试 (Playwright)              │
│  完整用户流程，真实浏览器                          │
├─────────────────────────────────────────────────┤
│              集成测试 (本文档)                     │
│  多模块协作，真实数据库，Mock 外部 API             │
├─────────────────────────────────────────────────┤
│              单元测试                             │
│  单模块/函数，Mock 所有依赖                       │
└─────────────────────────────────────────────────┘
```

### 1.2 集成测试范围

| 维度 | 说明 |
|------|------|
| 数据库 | 使用 PostgreSQL 测试实例（Docker） |
| Redis | 使用 Redis 测试实例 |
| Celery | 同步执行模式（`CELERY_ALWAYS_EAGER=True`） |
| 外部 API | Mock（LLM/支付/平台 API） |
| 文件系统 | 临时目录 |

### 1.3 测试环境

```bash
# 启动测试依赖
docker-compose -f docker-compose.test.yml up -d  # PG + Redis

# 运行集成测试
cd backend
pytest tests/integration/ -v --tb=short
```

## 6. 测试执行与 CI

### 6.1 本地执行

```bash
# 后端集成测试
docker-compose -f docker-compose.test.yml up -d
cd backend && pytest tests/integration/ -v

# 前端 E2E
cd frontend && npx playwright test

# 全量
make test-all
```

### 6.2 CI 流水线

```yaml
# .github/workflows/test.yml
stages:
  - unit-tests:      # 并行：后端 pytest + 前端 vitest
  - integration-tests: # 需要 PG + Redis Docker
  - e2e-tests:        # 需要完整服务启动 + Playwright
```
