## 1. 项目概述

### 1.1 背景

本平台整合 **AI 小说自动生成**（ai-novel-agent）与 **资讯/视频自动制作**（media-agent）两大核心能力，面向 C 端用户提供一站式"AI 内容创作 → 视频制作 → 多平台分发"的完整解决方案。平台通过抖音等渠道推广，以套餐付费模式运营。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| 小说自动生成 | 支持微小说、短篇、中篇、长篇、超长篇，涵盖儿童故事/男频/女频多种题材 |
| 视频自动制作 | 从 AI 小说、外部小说、资讯三种来源生成短视频 |
| 商业化运营 | 套餐付费 + 余额消耗，支持支付宝/微信支付 |
| 多平台分发 | 自动发布到抖音、小红书、番茄小说、起点等 |
| 开放接入 | 提供 API 与 OpenClaw 协议对接 |
| 多端入口 | App、微信/抖音小程序、龙虾（Lobechat）、Web |

### 1.3 现有系统概述

#### ai-novel-agent（当前能力）

- 7 Agent 协同流水线：热门趋势分析 → 风格解析 → 策划 → 写作 → 润色 → 审计 → 修订
- Web 端任务管理、进度监控、日志查看、文件下载
- 小说阅读平台 + 书架管理
- 支持测试模式递增章数至目标
- Docker/SSH 自动部署

#### media-agent（当前能力）

- RSS 资讯抓取（可配置栏目与源）
- 真人视频上传（多格式、预览）
- 生成流水线：DeepSeek 撰稿 → TTS 配音 → FFmpeg 合成 9:16 视频
- 发布适配器（当前为 Mock）
- 用户认证（注册/登录/图形验证码）
- 流程日志管理

## 2. 术语表

| 术语 | 定义 |
|------|------|
| **微小说** | 1000-5000 字极短篇小说，强冲突、快节奏 |

### 4.1 小说生成子系统

#### 4.1.1 小说类型与篇幅

| 编号 | 需求 | 优先级 |
|------|------|--------|
| NV-001 | 支持微小说生成（1000-5000 字），必须冲突感强 | P0 |
| NV-002 | 支持短篇小说生成（5000-30000 字），必须冲突感强 | P0 |
| NV-003 | 支持中篇小说生成（3-10 万字），必须冲突感强 | P0 |
| NV-004 | 支持长篇小说生成（10-50 万字） | P0 |
| NV-005 | 支持超长篇小说生成（50 万字以上） | P1 |

#### 4.1.2 题材分类

| 编号 | 需求 | 子分类 |
|------|------|--------|
| NV-010 | 儿童故事 | 0-3 岁启蒙、3-6 岁童话、6-9 岁冒险、9-12 岁成长、12+ 青少年 |
| NV-011 | 男频小说 | 玄幻、仙侠、军事、都市、科幻、历史、游戏、体育、悬疑、言情 |
| NV-012 | 女频小说 | 古代言情、现代言情、霸总、悬疑推理、纯爱、宫斗/宅斗、穿越重生、职场、甜宠、虐文 |
| NV-013 | 每个选项支持"随机"模式 | 系统自动从可选项中智能决策 |

#### 4.1.3 小说策划与创作规则

| 编号 | 需求 | 说明 |
|------|------|------|
| NV-020 | 参考网络大咖建议 | 系统内置各类型小说的创作建议库，每本小说策划时参照一个或多个不冲突的建议 |
| NV-021 | 策划唯一性保证 | 每本小说的策划方案经过去重校验，确保风格/结构/设定组合独一无二 |
| NV-022 | 冲突驱动叙事 | 微小说、短篇、中篇必须以强冲突为核心驱动力 |
| NV-023 | 视频适配增强 | 可生成视频的小说需增加记忆点设计，开头必须引人入胜 |
| NV-024 | 视频小说评审 | 生成视频的小说和文案必须经过 AI 审计 Agent 评审通过 |

#### 4.1.4 外部小说导入与抓取

| 编号 | 需求 | 说明 |
|------|------|------|
| NV-030 | 番茄小说抓取 | 从番茄小说平台抓取热门/指定小说 |
| NV-031 | URL 导入 | 用户提供小说 URL，系统自动抓取内容 |
| NV-032 | 全文导入 | 用户可直接粘贴/上传整部小说文本 |
| NV-033 | 其他平台扩展 | 预留起点、晋江等平台的抓取接口 |

#### 4.1.5 Agent 流水线（已有 + 增强）

|-------|------|------|

| StyleAgent | 风格解析，生成风格报告 | ✅ 已实现 |
| PlannerAgent | 策划大纲，参考大咖建议库 | 🔄 需增强 |
| WriterAgent | 章节写作 | ✅ 已实现 |
| PolishAgent | 润色 | ✅ 已实现 |
| AuditorAgent | 审计质检 | ✅ 已实现 |
| ReviserAgent | 修订 | ✅ 已实现 |
| ScorerAgent | 评分 | ✅ 已实现 |
| **VideoReviewAgent** | 视频小说专项评审 | 🆕 新增 |
| **ConflictAgent** | 冲突强度校验（微/短/中篇） | 🆕 新增 |

### 4.2 视频生成子系统

#### 4.2.1 视频来源模式

| 编号 | 需求 | 说明 |
|------|------|------|
| VD-001 | AI 小说 → 视频 | 从本平台 AI 生成的小说自动制作视频 |
| VD-002 | 外部小说 → 视频 | 用户导入的外部小说生成视频，可选部分篇幅（微/短/中）或全部随机 |
| VD-003 | 资讯 → 视频 | RSS/手动输入的资讯，经大模型归纳总结，口语化述说生成视频 |
| VD-004 | 资讯双语 | 资讯需保留原文版本及中文翻译版 |

#### 4.2.2 视频生成选项

| 编号 | 需求 | 说明 |
|------|------|------|
| VD-010 | 画面模式 — 语言生成画面 | AI 根据文本内容自动生成视频画面 |
| VD-011 | 画面模式 — 纯图片 | 仅使用 AI 生成的配图，无动态画面 |
| VD-012 | 画面模式 — 导入视频/音频 | 资讯类可使用用户自行导入的视频/音频素材 |
| VD-013 | 口型同步 | 视频播报时语音与口型保持一致 |
| VD-014 | 背景音乐 | 可选择是否添加背景音乐，支持音乐库选择 |
| VD-015 | 字幕控制 | 可选是否需要字幕；字幕样式、字体、颜色、位置可调 |
| VD-016 | 语音控制 | 可选是否需要语音；语速、语调、音色可选择 |
| VD-017 | 多选与冲突处理 | 以上选项支持多选，若存在冲突则随机决策 |
| VD-018 | 任务前确认/自动 | 每个任务生成前可让用户确认或选择自动模式 |

#### 4.2.3 视频生成流水线（已有 + 增强）

| 步骤 | 说明 | 状态 |
|------|------|------|
| 文本处理 | DeepSeek/Gemini 归纳总结，口语化改写 | ✅ 已实现（DeepSeek） |
| TTS 配音 | OpenAI TTS / Edge TTS，支持语速语调音色选择 | 🔄 需增强 |
| 画面生成 | Veo/Sora/可灵 生成视频画面 或 AI 图片 | 🆕 新增 |
| 口型同步 | VideoRetalk 口型一致处理 | ✅ 已实现 |
| 字幕烧录 | FFmpeg 字幕合成，支持样式自定义 | 🔄 需增强 |
| 背景音乐 | 音频混合 | ✅ 已实现 |
| 视频合成 | FFmpeg 最终合成 9:16 竖版视频 | ✅ 已实现 |

### 7.2 media-agent 现有能力 vs 新需求

| 需求领域 | 当前状态 | 差距 |
|----------|----------|------|
| 资讯抓取 → 视频 | ✅ RSS + DeepSeek + TTS + FFmpeg | 需增强口语化与双语 |
| 视频上传与管理 | ✅ 多格式上传预览 | 无 |
| TTS 配音 | ✅ OpenAI TTS + Edge TTS | 需增加语速语调音色选择 |
| 口型同步 | ✅ VideoRetalk | 无 |

| 字幕 | ⚠️ 基础字幕 | 需增加样式/字体/颜色自定义 |
| Mock 发布 | ✅ 模拟发布 | 需对接真实平台 API |

| 付费系统 | ❌ 未实现 | 需全新开发 |
| 画面生成（AI 视频） | ❌ 仅合成 | 需接入 Veo/Sora/可灵 |
| 多端入口 | ❌ 仅 Web | 需新增 App/小程序 |

### 7.3 整合结论

两个项目各有侧重，**ai-novel-agent 擅长小说生成**，**media-agent 擅长视频制作**。需求变更要求将两者整合为统一平台，主要差距集中在：

1. **商业化基础设施**：付费系统、套餐管理、支付对接（两个项目均缺失）
2. **用户体系升级**：社交登录、多端认证（media-agent 有基础，需升级）
3. **跨系统集成**：小说生成 → 视频制作的打通（两个项目需桥接）
4. **多端入口**：App/小程序/LobeChat（两个项目均缺失）
5. **发布生态**：真实平台对接（media-agent 有 Mock 基础，需真实对接）
6. **管理后台与官网**：全新开发
