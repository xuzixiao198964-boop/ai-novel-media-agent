## 1. 测试策略

### 1.1 测试框架

| 端 | 框架 | 说明 |
|------|------|------|
| 后端 | pytest + pytest-asyncio | 异步测试支持 |
| 用户端 Web | Vitest + Testing Library | React 组件测试 |
| 后台管理 Web | Vitest + Testing Library | React 组件测试 |
| App (React Native) | Jest + React Native Testing Library | 移动端测试 |
| 前端静态契约 | Node.js 脚本 | 零依赖契约检查 |

### 1.2 覆盖率目标

| 模块 | 行覆盖率目标 |
|------|------------|
| 核心业务逻辑（Agent/引擎） | ≥ 85% |
| API 路由层 | ≥ 80% |
| 服务层 | ≥ 80% |
| 工具函数 | ≥ 90% |
| 前端组件 | ≥ 70% |
| 前端页面 | ≥ 60% |

### 1.3 测试约定

- 使用临时目录/内存数据库，不污染生产数据
- Mock 外部 API（LLM、支付、平台 API）
- 异步测试使用 `asyncio_mode=auto`
- 前端测试使用 jsdom 环境

### 4.1 conftest.py 配置

```python
@pytest.fixture
def test_db():
    """使用 SQLite 内存数据库"""
    engine = create_engine("sqlite:///:memory:")
    Base.metadata.create_all(engine)
    yield SessionLocal(bind=engine)

@pytest.fixture
def test_user(test_db):
    """创建测试用户"""
    user = User(username="testuser", email="test@test.com", balance=100.0)
    test_db.add(user)
    test_db.commit()
    return user

@pytest.fixture
def mock_llm():
    """Mock LLM API 响应"""
    with patch("app.services.ai_text.call_llm") as mock:
        mock.return_value = "Mock LLM response"
        yield mock
```
