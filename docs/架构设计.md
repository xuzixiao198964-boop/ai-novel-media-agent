## 2. 模块架构

### 2.1 小说生成引擎

**7 Agent 协同流水线（已实现）**

```
┌─────────────────────────────────────────────────────────────┐
│                     小说生成引擎                              │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              Agent 流水线 (Pipeline)                  │  │
│  │                                                       │  │
│  │  TrendAgent (趋势分析，18种题材)                       │  │
│  │       ↓                                               │  │
│  │  StyleAgent (风格解析)                                 │  │
│  │       ↓                                               │  │
│  │  PlannerAgent (策划案 → 故事总纲 → 章节大纲)           │  │
│  │       ↓                                               │  │
│  │  WriterAgent (逐章创作，含前两章引子)                   │  │
│  │       ↓                                               │  │
│  │  PolishAgent (章节润色)                                │  │
│  │       ↓                                               │  │
│  │  AuditorAgent (质量审核，每3章一次)                     │  │
│  │       ↓                                               │  │
│  │  ReviserAgent (修订发布)                               │  │
│  └───────────────────────────────────────────────────────┘  │
│                              │                              │
│                        ┌─────┴─────┐                       │
│                        │ 记忆库管理  │                       │
│                        │ (任务目录)  │                       │
│                        └───────────┘                       │
└─────────────────────────────────────────────────────────────┘
```

**核心特性**：

1. **TrendAgent - 趋势分析**
   - 18种题材类型（儿童6种、男频6种、女频6种）
   - 动态章节建议（30-500章）
   - 动态字数建议（500-5000字/章）
   - 主题去重（3次内不重复）

2. **StyleAgent - 风格解析**
   - 叙事节奏分析
   - 语言风格定义
   - 情感基调设定

3. **PlannerAgent - 策划与大纲**
   - 策划案生成（核心设定、人设矩阵、故事线、章节规模）
   - 故事总纲（每5章一批，累计3批审核一次）
   - 章节大纲（每章含标题、事件、钩子、衔接）
   - 多轮审核机制（强制放行阈值=2）

4. **WriterAgent - 正文创作**
   - **前两章引子**：从第3章起传入第1、2章全文
   - **上章衔接**：引用上一章后半段原文
   - **总纲对齐**：提取当前章节对应的总纲段落
   - **角色名一致性**：严格按策划案人设矩阵
   - **字数保证**：每章不少于配置字数（默认10000字）

5. **PolishAgent - 章节润色**
   - 优化语言表达
   - 修正语法错误
   - 增强可读性

6. **AuditorAgent - 质量审核**
   - **单章打分**：每章写完后立即打分（70分及格）
   - **区间审计**：每3章进行一次区间审核
   - **审核维度**：逻辑连贯性、大纲符合度、人物一致性、伏笔回收
   - **智能重试**：根据分数趋势决定是否继续重写

7. **ReviserAgent - 修订发布**
   - 根据审核意见最终修订
   - 输出成书文件（含目录可跳转）
   - 自动同步到小说阅读平台

**质量保障机制**：

- **前两章引子机制**：确保角色性格、语言风格、世界观一致
- **智能打分系统**：单章打分 + 区间审核双重保障
- **测试模式递增**：6→18→54→162...，逐步验证至目标章数

   - 分辨率：9:16竖屏（适配抖音/快手）
   - 字幕烧录：自动生成、样式可定制
   - 背景音乐：可选、音量可调
   - 口型同步：AI驱动的唇形匹配

### 4.1 API 路由规划

| 前缀 | 服务 | 说明 |
|------|------|------|
| `/` | 产品官网 | 静态页面（port 80） |
| `/api/v1/auth` | 用户认证 | 注册/登录/OAuth/刷新 |

## 9. 部署架构改进方案（2026-04-05新增）

### 9.1 当前部署问题分析

#### 问题1：SSH连接不稳定
**现象**：
- SSH认证失败（Authentication failed）
- ICMP被阻止（ping不通）
- HTTP服务正常

**影响**：
- 无法上传代码
- 无法重启服务
- 阻塞部署流程

**根因分析**：
1. 服务器防火墙配置：允许HTTP但阻止SSH/ICMP
2. 可能的密码更改
3. fail2ban等安全机制触发

#### 问题2：部署依赖单一通道
**现象**：
- 仅依赖SSH进行部署
- SSH失败则部署完全阻塞

**影响**：
- 部署脆弱性高
- 无备用方案

### 9.2 改进方案

#### 方案1：多通道部署架构

#### 方案3：部署前置检查

```python
class DeploymentValidator:
    """部署前置检查"""
    
    async def validate_all(self) -> ValidationResult:
        checks = [
            self.check_ssh_connection(),
            self.check_disk_space(),
            self.check_dependencies(),
            self.check_service_status(),
            self.check_database_connection(),
        ]
        results = await asyncio.gather(*checks)
        return ValidationResult(results)
    
    async def check_ssh_connection(self):
        """检查SSH连接"""
        try:
            client = paramiko.SSHClient()
            client.connect(host, username, password, timeout=10)
            return CheckResult(passed=True)
        except Exception as e:
            return CheckResult(passed=False, error=str(e))
```
